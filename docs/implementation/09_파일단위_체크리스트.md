# 09. 파일 단위 체크리스트 (레이어별 인덱스)

- 최종 수정일: 2026-04-17
- 관련 스펙: `../specs/07_기술스택_프로젝트구조.md` 전체
- 목적: 마일스톤 문서에서 참조하는 **모든 생성 파일의 인덱스**. 각 파일마다 `목적 / 주요 exports & 시그니처 / 의존성 / 테스트 포인트` 4단 기술.

## 전체 디렉토리 트리

```
playwright-hub/
├── apps/
│   ├── web/                          # Next.js 14 App Router (포트 3000)
│   └── api/                          # Fastify 5 + fastify-sse-v2 + BullMQ Worker (포트 3001)
├── packages/
│   └── shared-types/                 # WS 이벤트 / API 스키마 공유
├── docker/
│   └── playwright-runner/            # Playwright 실행 컨테이너
├── deploy/                           # Docker Compose / worker-compose / 스크립트
├── prisma/                           # schema.prisma + migrations + seed
├── projects/                         # runtime: stable/ working/ 디렉토리
├── reports/                          # runtime: run-id별 리포트
├── tests/
│   └── e2e/                          # Playwright E2E
├── scripts/                          # 유틸(sdk-spike, docker build 등)
├── docs/
│   ├── specs/                        # 기존 명세
│   └── implementation/                # 본 문서 집합
├── .github/workflows/ci.yml
└── 루트 설정 파일들
```

## 2. Prisma

### 2.1 `prisma/schema.prisma`
- **목적**: PostgreSQL 스키마.
- **exports**: 모델 `Org`, `User`, `Project`, `Run`, `Edit` + enum `Role`, `RunStatus`, `EditStatus`.
- **주요 필드**:
  - `User.email @unique`, `@@index([orgId])`
  - `Project.@@unique([orgId, name])`, `workerGroup String?`, `config Json`
  - `Run.@@index([projectId, createdAt DESC])`, `@@index([status])`, `@@index([userId])`
  - `Edit.@@index([projectId, createdAt DESC])`, `sessionId`, `messages Json @default("[]")`, `costUsd Float?`, `durationMs Int?`
  - Cascade: Project→Org, Run→Project, Edit→Project
- **테스트 포인트**: `prisma validate` + `prisma migrate dev` 성공.

### 2.2 `prisma/seed.ts`
- **목적**: Demo Org + Admin 계정 + (선택) 샘플 프로젝트 upsert.

### 2.3 `prisma/migrations/<timestamp>_init/migration.sql`
- **목적**: 초기 마이그레이션 (CLI 생성).

## 3. API 백엔드 (`apps/api/src/`)

### 3.1 `index.ts`
- **목적**: Fastify 앱 부팅.
- **흐름**: dotenv → config 검증 → `fastify({ logger: pinoConfig })` → 플러그인 등록(`@fastify/cors`, `@fastify/helmet`, `@fastify/compress`, `@fastify/sensible`, `@fastify/jwt`, `@fastify/type-provider-zod`, `fastify-sse-v2`) → 라우트 등록(`/api/v1/auth` 무인증 + `/api/v1/*` authHook 적용) → `setErrorHandler(errorHandler)` → `registerRunSse`/`registerEditSse` 플러그인 등록 → `listen({ port: config.PORT, host: '0.0.0.0' })` → SIGTERM 핸들러.

### 3.2 `lib/config.ts`
- **목적**: Zod env 검증 (Single Source of Truth).
- **exports**: `configSchema`, `config`, `type Config`.
- **테스트 포인트**: 필수 변수 누락 시 부팅 실패.

### 3.3 `lib/prisma.ts`
- **목적**: PrismaClient 싱글톤 (HMR 대응).
- **exports**: `prisma`.

### 3.4 `lib/redis.ts`
- **목적**: ioredis 팩토리 + pub/sub 분리 클라이언트.
- **exports**: `createRedisConnection(url?)`, `redis`, `pubClient`, `subClient`.

### 3.5 `lib/queue.ts`
- **목적**: BullMQ Queue/QueueEvents 팩토리.
- **exports**: `createRunsQueue`, `createRunsQueueEvents`.

### 3.6 `lib/logger.ts`
- **목적**: pino 로거.
- **exports**: `logger`, `httpLogger`.

### 3.7 `lib/errors.ts`
- **목적**: 도메인 에러 계층.
- **exports**: `AppError`, `ValidationError`, `UnauthorizedError`, `ForbiddenError`, `NotFoundError`, `ConflictError`, `RateLimitError`, `InternalError`.

### 3.8 `plugins/auth.ts`
- **목적**: JWT 검증 Fastify 플러그인.
- **exports**: `authPlugin` (`@fastify/jwt` 래핑), `authHook(request, reply)` preHandler, `signToken(payload)`, `verifyToken(token)`.

### 3.9 `hooks/orgScope.ts`

### 3.10 `plugins/validation.ts`

- **exports**: `validationPlugin`, 필요 시 `validateRequest(schemas)` preHandler 헬퍼.

### 3.11 `plugins/errorHandler.ts`
- **목적**: `fastify.setErrorHandler(handler)` 등록 플러그인.
- **exports**: `registerErrorHandler(fastify)` 함수. AppError 계열을 `{ success, error: { code, message } }` 포맷으로 변환.

### 3.12 `plugins/requestId.ts`
- **목적**: correlation id(x-request-id) 주입 `onRequest` hook. Fastify 내장 `request.id`와 연계.

## 11. 사용 방법

- 각 마일스톤 문서(`02_~08_`)에서 해당 마일스톤의 파일 체크박스를 따라 진행
- 이 문서는 **"어떤 파일이 어느 경로에 있어야 하는지"의 글로벌 인덱스**로 활용
- 파일 생성 시 기본 템플릿/스켈레톤은 해당 마일스톤 문서에서 참조
