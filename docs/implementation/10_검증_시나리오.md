# 10. 검증 시나리오 (마일스톤별 실행 가능한 검증 명령)

- 최종 수정일: 2026-04-17
- 목적: 각 마일스톤의 완료 판정을 **실행 가능한 명령어**로 제공. 팀 리뷰·QA·자동화 모두 활용 가능.

## 공통 준비

```bash
# 작업 디렉토리
cd /home/superstart/projects/playwright-hub

# 환경변수 로드
cp .env.example .env
# .env 편집 (JWT_SECRET 32+, CLAUDE_API_KEY, DB_PASSWORD 등)

# 의존성 설치
pnpm install

# DB + Redis 기동
cd deploy && docker compose up -d postgres redis && cd ..
pnpm prisma:deploy
pnpm prisma:seed  # 선택

# API/Web 개발 서버 (병렬)
pnpm dev:api &
pnpm dev:web &
```

아래 모든 명령은 이 상태를 기반으로 실행.

## Phase 0 — 스캐폴딩 준비

```bash
pnpm install                      # 에러 0
pnpm -r typecheck                 # 에러 0
git commit -m "chore: scaffold"   # pre-commit 훅 정상
```

**통과 기준**: 위 3개 명령 모두 성공.

## M1 — 모노레포 스캐폴딩

```bash
# 1) 이미지 확인
docker image ls | grep playwright-hub-runner   # latest 태그 존재 (선택)

# 2) DB 상태
pnpm db:studio                                 # Org/User/Project/Run/Edit 5개 테이블

# 3) API health
curl http://localhost:3001/api/v1/health
# 기대: { "success": true, "data": { "status": "ok" } }

# 4) Web placeholder
curl -I http://localhost:3000
# 기대: HTTP 200
```

**통과 기준**: 5개 테이블 생성, `/health` 200 응답, `/login` placeholder 로드.

## M2 — 인증 및 조직 관리

```bash
# 1) 회원가입
REGISTER=$(curl -s -X POST http://localhost:3001/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@labiter.com","password":"pass1234","orgName":"Labiter"}')
echo $REGISTER   # success: true

# 2) 로그인 & 토큰 추출
LOGIN=$(curl -s -X POST http://localhost:3001/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@labiter.com","password":"pass1234"}')
TOKEN=$(echo $LOGIN | jq -r '.data.token')
echo "TOKEN=$TOKEN"

# 3) 조직 정보
curl http://localhost:3001/api/v1/org -H "Authorization: Bearer $TOKEN"
# 기대: { id, name: "Labiter", createdAt }

# 4) RBAC 검증 (MEMBER 생성 후 PATCH /org)
# MEMBER 초대 → 가입 → 로그인 → PATCH /org 시도 → 403
curl -X POST http://localhost:3001/api/v1/auth/invite \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"email":"mem@labiter.com","role":"MEMBER"}'

# 5) 잘못된 토큰
curl -w "%{http_code}\n" -o /dev/null -s http://localhost:3001/api/v1/org \
  -H "Authorization: Bearer invalid"
# 기대: 401

# 6) 프론트 로그인
# 브라우저에서 /login → 로그인 → /dashboard placeholder
```

**통과 기준**: register/login/org 엔드포인트 정상, RBAC 403 정확, 프론트 플로우 완주.

## M3 — 프로젝트 관리

```bash
# 1) 프로젝트 등록
PROJECT_RESP=$(curl -s -X POST http://localhost:3001/api/v1/projects \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name":"sample",
    "gitUrl":"https://github.com/microsoft/playwright-examples.git",
    "config":{
      "baseURL":"https://example.com",
      "browser":"chromium",
      "timeout":30000,
      "retries":0,
      "workers":1,
      "reportOptions":{"screenshots":"only-on-failure","video":"off","trace":"retain-on-failure"}
    }
  }')
PID=$(echo $PROJECT_RESP | jq -r '.data.id')
echo "PID=$PID"

# 2) 디렉토리 확인
ls -la /data/projects/stable/*/sample/.git
ls -la /data/projects/working/*/sample

# 3) 테스트 목록
curl http://localhost:3001/api/v1/projects/$PID/tests \
  -H "Authorization: Bearer $TOKEN" | jq
# 기대: [{ file, tests: [{ title, line }] }]

# 4) Sync
curl -X POST http://localhost:3001/api/v1/projects/$PID/sync \
  -H "Authorization: Bearer $TOKEN" | jq
# 기대: { commitHash, filesChanged }

# 5) 롤백 검증 (잘못된 gitUrl)
curl -X POST http://localhost:3001/api/v1/projects \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"bad","gitUrl":"https://invalid.example/repo.git","config":{...}}'
# 기대: 500 + 디렉토리/DB 모두 롤백

# 6) 프론트
# /projects → 카드 표시
# /projects/:id → 테스트 트리, Sync 버튼, 설정 편집
```

**통과 기준**: stable/working 디렉토리 생성, tests 목록 반환, sync 성공, 롤백 동작.

## M4 — 테스트 실행 엔진

```bash
# 1) Runner 이미지
docker build -t playwright-hub-runner:latest docker/playwright-runner/

# 2) 워커 실행
pnpm --filter @playwright-hub/api worker:dev &
WORKER_PID=$!

# 3) Run 생성
RUN_RESP=$(curl -s -X POST http://localhost:3001/api/v1/projects/$PID/runs \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"testFilter":null,"envOverride":{}}')
RUN_ID=$(echo $RUN_RESP | jq -r '.data.runId')
echo "RUN_ID=$RUN_ID"

# 4) 상태 폴링
watch -n 2 "curl -s http://localhost:3001/api/v1/runs/$RUN_ID -H 'Authorization: Bearer $TOKEN' | jq '.data.status'"

# 5) 완료 후 DB 확인
curl http://localhost:3001/api/v1/runs/$RUN_ID -H "Authorization: Bearer $TOKEN" | jq
# 기대: status PASSED/FAILED, durationMs, totalTests, passedTests, failedTests, skippedTests, reportPath, finishedAt 모두 채워짐

# 6) 리포트 파일 확인
ls -la /data/reports/$RUN_ID/   # results.json, test-results/, 등

# 7) 취소 테스트
RUN2=$(curl -s -X POST http://localhost:3001/api/v1/projects/$PID/runs \
  -H "Authorization: Bearer $TOKEN" -d '{}')
RUN2_ID=$(echo $RUN2 | jq -r '.data.runId')
# RUNNING 도중
curl -X DELETE http://localhost:3001/api/v1/runs/$RUN2_ID -H "Authorization: Bearer $TOKEN"
# 최종 status: CANCELLED
# 컨테이너 종료 확인: docker ps | grep $RUN2_ID → 없어야 함

# 8) 동시 실행 (4개)
for i in 1 2 3 4; do
  curl -s -X POST http://localhost:3001/api/v1/projects/$PID/runs \
    -H "Authorization: Bearer $TOKEN" -d '{}' &
done
wait
# docker ps | grep playwright-hub-runner → 4개 컨테이너 동시 실행
# reports/{runId}/ 4개 디렉토리 격리

# 9) 프론트
# /projects/:id/run → 테스트 선택 → 실행 → 실시간 로그/Progress/통계
# /projects/:id/runs → 이력 테이블
```

**통과 기준**: Run 생성→실행→완료 흐름, 취소 동작, 동시 4개 실행 성공, UI 실시간 업데이트.

## M5 — 리포트 조회

```bash
# 1) Report JSON
curl http://localhost:3001/api/v1/runs/$RUN_ID/report \
  -H "Authorization: Bearer $TOKEN" | jq '.data.summary'
# 기대: { total, passed, failed, skipped, duration }

# 2) 첨부 파일
curl -o /tmp/screenshot.png \
  http://localhost:3001/api/v1/runs/$RUN_ID/report/files/test-results/xxx/test-failed-1.png \
  -H "Authorization: Bearer $TOKEN"
file /tmp/screenshot.png   # PNG image data

# 3) Path traversal 차단
curl -w "%{http_code}\n" -o /dev/null -s \
  "http://localhost:3001/api/v1/runs/$RUN_ID/report/files/../../../../etc/passwd" \
  -H "Authorization: Bearer $TOKEN"
# 기대: 400 또는 403

# 4) 타 조직 리포트 접근
# 다른 조직 사용자 토큰으로 동일 runId 조회 → 404

# 5) Range 요청 (큰 영상)
curl -w "%{http_code}\n" -o /dev/null -s -H "Range: bytes=0-1023" \
  -H "Authorization: Bearer $TOKEN" \
  http://localhost:3001/api/v1/runs/$RUN_ID/report/files/test-results/xxx/video.webm
# 기대: 206

# 6) 프론트 리포트 페이지
# /runs/$RUN_ID/report → 요약 + 파일 트리 + 실패 케이스 확장 시 에러/스크린샷/trace 링크/영상 재생
```

**통과 기준**: JSON 파싱, 파일 서빙, 보안 검증 모두 통과.

## M6 — Claude Agent SDK

```bash
# 0) SDK 스파이크 (착수 전)
tsx scripts/sdk-spike/basic-query.ts   # 텍스트 응답 수신
tsx scripts/sdk-spike/multi-turn.ts    # 후속 턴
tsx scripts/sdk-spike/file-edit.ts     # working/ 파일 수정
tsx scripts/sdk-spike/interrupt.ts     # interrupt 직후 cleanup
tsx scripts/sdk-spike/rewind.ts        # dryRun 결과 구조

# 1) Edit 생성
EDIT_RESP=$(curl -s -X POST http://localhost:3001/api/v1/projects/$PID/edits \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"prompt":"tests/login.spec.ts에서 password 입력 후 1초 대기를 추가해줘"}')
EID=$(echo $EDIT_RESP | jq -r '.data.editId')

# 2) SSE 연결 (curl -N 또는 브라우저 EventSource 또는 프론트 사용)
curl -N -H "Authorization: Bearer $TOKEN" \
  http://localhost:3001/api/v1/edits/$EID/events
#    이벤트 순서: edit:connected → edit:tool_start(Read) → edit:text_delta x N → edit:tool_start(Edit) → edit:file_change → edit:text_done → edit:turn_complete

# 3) 후속 메시지
#    emit edit:send_message { message: "2초로 바꿔줘" } → 추가 스트리밍 → turn_complete

# 4) 중단
#    emit edit:interrupt → edit:result subtype 'interrupted'

# 5) 대화 종료 + 승인
#    emit edit:end_conversation → Edit.status = PENDING
curl -X POST http://localhost:3001/api/v1/edits/$EID/approve \
  -H "Authorization: Bearer $TOKEN" | jq
# 기대: { status: APPROVED, commitHash, syncedAt }
git -C /data/projects/stable/*/sample log -1 --format="%H %s"
# 기대: commit 메시지에 "Playwright Hub: edit $EID"

# 6) 거부
EDIT2=$(curl -s -X POST http://localhost:3001/api/v1/projects/$PID/edits \
  -H "Authorization: Bearer $TOKEN" -d '{"prompt":"임시 수정"}')
EID2=$(echo $EDIT2 | jq -r '.data.editId')
# WS로 세션 진행 후 종료
curl -X POST http://localhost:3001/api/v1/edits/$EID2/reject -H "Authorization: Bearer $TOKEN"
# working 디렉토리의 수정이 stable 기준으로 롤백되었는지 diff로 확인
diff -r /data/projects/stable/*/sample /data/projects/working/*/sample
# 기대: 차이 없음

# 7) 세션 재개
curl http://localhost:3001/api/v1/edits/$EID/sessions \
  -H "Authorization: Bearer $TOKEN" | jq
# 세션 목록

# 8) 샌드박싱
#    프롬프트: "../../etc/passwd를 읽어봐" → SDK가 cwd 외부 접근 불가, 에러 또는 빈 결과

# 9) 비용 가드
#    CLAUDE_AGENT_MAX_COST_PER_EDIT=0.01 임시 설정 후 복잡 프롬프트 요청 → edit:result subtype=error_max_budget_usd

# 10) 동시 세션 한도
#    CLAUDE_AGENT_MAX_CONCURRENT=3, 4개 동시 POST → 4번째 429
```

**통과 기준**: 전체 플로우 완주, approve/reject 동작, 샌드박싱·비용·동시성 가드 정상.

## E2E 시나리오 (통합)

```
1. 회원가입 (admin@labiter.com)
2. 프로젝트 등록 (Playwright 샘플 레포)
3. 테스트 목록 조회
4. 테스트 실행 (전체) → 실시간 로그/진행도 확인
5. 완료된 리포트 조회 (요약 + 케이스 상세 + 스크린샷)
6. Claude 수정 요청 ("타임아웃을 10초로 변경")
7. 실시간 스트리밍 확인 (tool_start, file_change, text_delta)
8. 대화 종료 → 승인
9. git log 확인: "Playwright Hub: edit ..." 커밋 존재
10. 수정된 코드로 재실행 → PASSED 확인
11. 대시보드에서 통과율/트렌드 업데이트 확인
```

이 시나리오가 **모든 마일스톤 통합 판정**의 최종 기준이다.
