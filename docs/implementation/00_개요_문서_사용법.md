## 1. 이 문서 집합은 무엇인가

`docs/specs/`의 8개 명세 문서를 기반으로, Playwright Hub 프로젝트를 **greenfield 레포에서 Phase 3 확장까지** 실제로 구현하기 위한 단계별·파일 단위 작업 계획이다. 명세가 "무엇을 만들 것인가"라면, 본 문서 집합은 "어떤 순서로, 어떤 파일을, 어떤 기준으로 만들 것인가"를 다룬다.

## 2. 독자

## 6. 핵심 용어 정의

| 용어 | 정의 |
|------|------|
| **stable/** | Docker가 읽기 전용(`:ro`)으로 마운트하여 테스트를 실행하는 안정 버전 디렉토리. 경로: `${PROJECTS_STABLE_PATH}/{projectPath}` |
| **working/** | Claude Agent SDK가 파일 편집을 수행하는 작업 디렉토리. 경로: `${PROJECTS_WORKING_PATH}/{projectPath}` |
| **Run** | 한 번의 테스트 실행 단위. 고유 `runId`, 상태 전이 `QUEUED→RUNNING→PASSED/FAILED/CANCELLED` |
| **Edit** | Claude Agent SDK의 한 번의 수정 요청 단위. 상태 전이 `STREAMING→WAITING_INPUT→PENDING→APPROVED/REJECTED` |
| **Worker** | BullMQ 큐에서 작업을 꺼내 Docker 컨테이너를 실행하는 Node.js 프로세스 |
| **Concurrency** | 하나의 Worker 프로세스가 동시에 처리할 수 있는 작업 수 (기본값: 4) |
| **Queue Router** | API 서버에서 작업을 적절한 Redis/큐에 분배하는 라우팅 모듈. 단일→다중 Redis 전환의 추상화 계층 |
| **동기화(sync)** | Edit 승인 시 `working/` 내용을 `stable/`에 반영하고 Git commit 하는 작업 |
| **Edit 세션** | `@anthropic-ai/claude-agent-sdk`의 `query()`가 만드는 AsyncGenerator 수명. sessionId로 재개 가능 |

## 7. 구현 시 지키는 10개 원칙

1. **스펙 → 구현 일관성**: 구현 중 스펙과 모순 발견 시 스펙을 먼저 PR로 수정한다.
2. **레이어 분리**: `route → service → db/external`. 라우트는 서비스만, 서비스는 Prisma/외부만 호출.
3. **zod-first**: 모든 외부 입력(HTTP body·query·params, env, WS message)은 Zod 스키마로 선검증.
4. **조직 격리 불변조건**: 모든 Prisma where 절에 `orgId` 포함. 누락 방지를 위해 `orgScope` 미들웨어 + 서비스 레이어 이중 체크.
5. **에러 규격화**: API 응답은 항상 `{ success, data | error: { code, message } }` 포맷. 내부 스택은 로그에만.
6. **WS 이벤트 타입 공유**: `packages/shared-types` 또는 유사 구조로 서버/클라이언트 이벤트 타입 단일화.
7. **원자적 상태 전이**: Run/Edit 상태 변경은 `prisma.$transaction` + 분산 락과 함께.
8. **보안 기본값**: Docker 컨테이너 리소스 제한, path traversal 방지, Claude SDK `cwd` 샌드박싱.
9. **관찰성**: 모든 비동기 작업에 correlation id (`runId`, `editId`, `request-id`) 부여.
10. **배포 재현성**: 매 마일스톤 종료 시 clean state에서 `docker compose up -d`로 기동 검증.

## 8. 본 문서 작성 규칙

- 언어: 한국어 + Markdown (GFM).
- 파일 경로: 가능한 절대경로 사용. 스펙 문서 참조는 상대경로 링크.
- 검증 명령: 실제 실행 가능한 형태로 제시 (placeholder는 `$TOKEN`, `$PID`, `$RUN_ID`, `$EID`).
- 이모지: 사용하지 않음.
- 각 문서 상단에 "최종 수정일 + 관련 스펙" 메타 정보 필수.
- 상태 전이·의존성은 mermaid 다이어그램으로 표기.
